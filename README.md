# 电报机器人
Telegram 双向私聊机器人
一个基于 Cloudflare Workers 的 Telegram 机器人，实现用户与管理员之间的双向私密对话，利用群组话题（Topics） 为每个用户创建独立线程，支持文字和图片。

✨ 功能特点
用户侧：私聊机器人发送文字/图片 → 机器人自动在指定话题群组中创建/使用该用户的独立话题，仅转发原始内容（无任何前缀或提示）。

管理员侧：在话题中回复用户 → 机器人将回复（文字/图片）转发回原用户，无多余提示。

话题标题：自动格式化为 用户ID (姓名)，方便识别。

自动故障恢复：若话题被删除，用户下次发消息时自动重建。

灵活设置话题群：支持两种方式指定用于存放话题的群组：

自动检测：将机器人加入已开启话题的超级群组，机器人会向管理员发送带按钮的消息，一键同意即可。

手动命令：管理员私聊发送 /setchat 群ID 直接指定。

管理命令（私聊机器人）：

/setchat <群ID> – 手动设置话题群（需群已开启话题且机器人在群中）

/topicgroup – 查看当前设置的话题群

/cleartopicgroup – 清除话题群设置

📦 环境变量
在 Cloudflare Worker 中配置以下生产环境变量：

变量名	说明	示例
BOT_TOKEN	Telegram Bot Token	123456:ABCdef...
ADMIN_ID	管理员 Telegram 用户 ID（多个用英文逗号分隔）	123456789 或 123456,789012
RELAYGO_KV	KV 命名空间绑定（见下方部署）	–
注意：不再需要 DEFAULT_GROUP_ID，话题群通过命令或自动检测设置。

🚀 部署步骤
1. 创建 Bot 并获取 Token
联系 @BotFather，创建新机器人，获取 BOT_TOKEN。

获取你自己的用户 ID（可联系 @userinfobot）。

2. 创建 Cloudflare Worker 和 KV
登录 Cloudflare，进入 Workers 和 Pages。

创建一个新的 Worker（例如 tg-topic-bot）。

在 KV 页面创建一个命名空间（例如 tg_bot_kv），记下 ID。

3. 添加环境变量和 KV 绑定
进入 Worker → 设置 → 变量 → 生产环境：

添加 BOT_TOKEN 和 ADMIN_ID 为纯文本。

在 KV 命名空间绑定 中添加：

变量名称：RELAYGO_KV

选择你创建的 KV 命名空间。

4. 粘贴代码
将worker.js的代码复制到 Worker 编辑器中，完全替换默认内容。

点击 保存并部署。

5. 设置 Webhook
在浏览器中访问：
https://你的Worker域名/setwebhook

返回 {"ok":true,...} 表示成功。

6. 设置话题群
方法一（自动）：将机器人加入一个已开启话题的超级群组（群 ID 以 -100 开头，群设置中开启“话题”），管理员会收到一条带按钮的消息，点击“同意设置”即可。

方法二（手动）：管理员私聊机器人，发送 /setchat -1001234567890（替换为真实的群 ID）。需要确保机器人已在群中且群已开启话题。

🧪 使用示例
用户：私聊机器人发送“你好”。
→ 群组中自动创建话题 123456 (张三)，话题内显示“你好”。

管理员：进入该话题，回复“有什么事？”
→ 用户收到消息“有什么事？”。

管理员：在私聊中使用 /topicgroup 查看当前设置的话题群。
❗ 注意事项
群组必须为超级群组（ID 以 -100 开头）且已开启话题功能，否则无法创建话题。

机器人必须是群组管理员，并拥有“创建话题”和“发送消息”权限。
📋 命令列表
管理员命令（私聊机器人）
命令	             格式	                       说明
/setchat	        /setchat <群ID>	        手动设置话题群。群ID 必须是已开启话题的超级群组（以 -100 开头），且机器人已在群中。示例：/setchat -1001234567890
/topicgroup     	/topicgroup	            查看当前设置的话题群名称和 ID。
/cleartopicgroup	/cleartopicgroup	      清除话题群设置。清除后用户消息无法转发，需要重新设置话题群。
/start	          /start	                显示帮助信息。如果已设置话题群，提示机器人已启动；未设置时提示管理员进行配置。
普通用户命令
命令	              格式	                     说明
/start	          /start	                显示欢迎信息。机器人已设置话题群时，告知用户可用；未设置时提示联系管理员。

Webhook 默认配置了 allowed_updates，会自动接收 chat_member 事件（用于自动检测加入群组）。

所有映射（话题 ↔ 用户）存储在 KV 中，有效期 30 天，自动续期。
